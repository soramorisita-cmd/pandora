## 2. 画像表示・青文字バグの修正

### 問題
- 静的ページ（`/products/xxx.html`）で画像が表示されない
- カードのタイトルテキストが青くなる

### 原因と修正

| 問題 | 原因 | 修正 |
|------|------|------|
| 画像が出ない | `images/xxx.jpg` という相対パスが `/products/` フォルダから `/products/images/xxx.jpg` に解決される | `abs_img()` 関数を追加して `images/xxx` → `/images/xxx` に変換 |
| 青文字 | `card_css()` に `a{color:inherit}` がなかった | `card_css()` の先頭に追加 |

### 追加したコード（build_static.py）
```python
def abs_img(img):
    """相対パス 'images/xxx' を '/images/xxx' に変換する"""
    if img and img != "null" and not img.startswith(("http", "/")):
        return "/" + img
    return img or ""
```

## 4. 致命的バグの修正（catalog.html 読み込み中が消えない）

### 原因
`cleanTitle()` を置き換えるPythonスクリプトで、正規表現 `(?=\nconst )` が関数内部の `const stripped` に引っかかって途中で止まった。

結果: 古い関数の末尾コードが `}` の後に残留し、JavaScriptの構文エラーが発生。`init()` が一切実行されず、ローディングスピナーが永遠に消えない状態になった。

### 修正
壊れたブロックを `re.search(r'function cleanTitle\(r\)\{.*?(?=\nfunction extractJPY)', content, re.DOTALL)` で正確に特定し、クリーンな実装で完全置換。

### 教訓
- JSファイルをPythonで部分置換するとき、**関数内に置換パターンと同じキーワード（`const`、`function` 等）が含まれる場合は正規表現が誤マッチする**
- 安全な方法: 関数全体を1つのユニークな文字列（関数名）で検索し、次の関数名を終端として使う
